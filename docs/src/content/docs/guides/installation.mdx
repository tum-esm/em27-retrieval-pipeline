**System requirements**

We can only guarantee that the pipeline works on Unix-systems. You need Python3.11+ installed on your system as well as `unzip` and `gfortran`.

**Install the pipeline**

1. Clone the [repository](https://github.com/tum-esm/em27-retrieval-pipeline) or download the source code of a [specific release](https://github.com/tum-esm/em27-retrieval-pipeline/releases).
2. Create a virtual environment using `python3.11 -m venv .venv`
3. Always activate the virtual environment using `source .venv/bin/activate`
4. Install the dependencies using `pip install ".[dev]"` or `pdm sync --with=dev`

<Aside type="tip">

We recommend using [PDM](https://pdm-project.org/) - a Python package manager - over pip. PDM uses a `pdm.lock` file - next to the `pyproject.toml` file - to pin the dependencies to specific versions. `pdm sync` installs the exact same dependencies on every system, hence errors are less likely not to be reproducible.

</Aside>

**Configure it on your system**

5. Install system dependencies: e.g. `sudo apt install unzip gfortran`
6. Use the `config/config.template.json` to create a `config.json` file.
7. Set up the metadata locally or remotely (see [metadata guide](/guides/metadata))

**Run the tests to ensure everything is working**

8. Test the pipeline using `pytest -m "integration or quick or ci" --verbose --exitfirst tests/`
9. Test the actual retrieval using `pytest -m "complete" --verbose --exitfirst tests/`

**Celebrate if the tests pass**

10. Run `curl parrot.live`
